# CS50x Week 1 Notes — C

## Main idea

Week 1 moves from Scratch to C. The ideas are similar to Scratch, but the syntax is stricter.

## Key concepts

### Source code
Source code is code written by humans.

### Machine code
Machine code is the 0s and 1s that computers understand.

### Compiler
A compiler translates source code into machine code.

### C
C is a programming language that gives more control over how computers work.

### VS Code
VS Code is the coding environment used in CS50.

### Terminal
The terminal is where commands are typed, such as compiling and running programs.

### make
`make` compiles a C program.

### ./program
`./program` runs a compiled program.

### Header files
Header files give access to functions written by others.

Examples:
- `#include <stdio.h>` gives access to `printf`
- `#include <cs50.h>` gives access to CS50 functions like `get_string`, `get_int`, and `get_char`

### Variables
Variables store values.

Examples:
- `int` stores integers
- `string` stores text
- `char` stores one character
- `float` stores decimal numbers
- `long` stores larger integers

### Conditionals
Conditionals let programs make decisions.

Examples:
- `if`
- `else if`
- `else`

### Loops
Loops repeat code.

Examples:
- `while`
- `for`
- `do while`

### Functions
Functions let us create reusable actions.

### Correctness, design, and style
Good code should:
- work correctly
- be well-designed
- be readable and clean

### Integer overflow
Integer overflow happens when a number becomes too large for the memory allocated to it.

### Floating-point imprecision
Floating-point imprecision happens because computers cannot perfectly represent all decimal numbers.
